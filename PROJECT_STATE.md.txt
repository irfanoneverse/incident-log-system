## Architecture Decisions

* **Framework**: Nuxt 4 (Nuxt 4.4.8) with Vue 3 and TypeScript strict mode.
* **Styling**: Tailwind CSS via `@nuxtjs/tailwindcss` module.
* **ORM**: Prisma 6 with PostgreSQL datasource.
* **Validation**: Zod ‚Äî single source of truth for all request validation.
* **Auth**: Cookie-based session (`HttpOnly`, `SameSite=lax`, 8-hour TTY).
  Credentials stored in env vars (`ADMIN_USERNAME`, `ADMIN_PASSWORD`).
* **ID type**: `cuid()` ‚Äî URL-safe, no UUID verbosity.
* **Prisma singleton**: `globalForPrisma` pattern to prevent multiple clients on hot-reload.
* **Server imports**: Nuxt 4 resolves `~` to `app/` (srcDir), not the project root. All `server/` files use relative imports for `server/lib/`, `server/validations/`, and `server/utils/`.

---

## Database Decisions

* Single model: `Incident`.
* Three enums: `IncidentCategory`, `IncidentPriority`, `IncidentStatus`.
* `resolvedAt` is managed automatically by the API layer (not manually set by client).
* Indexes on: `status`, `category`, `priority`, `createdAt`, `(status, createdAt)`.

---

## API Decisions

* All API routes under `/api/`.
* Auth routes at `/api/auth/*` ‚Äî public (no cookie required).
* All other `/api/*` routes protected by global server middleware.
* Incident list uses Prisma parallel `count` + `findMany` for performance.
* Dashboard stats fetched in a single `/api/dashboard/stats` request (5 parallel queries).

---

## UI Decisions

* **Auth composable**: `useAuth()` with `useState('auth.state')` ‚Äî single shared reactive state across the app.
* **Auth check strategy**: `null` = unknown (first load), `true` = authenticated, `false` = not authenticated. Fetches once per session; subsequent navigations skip the network call.
* **Login page**: `definePageMeta({ layout: false })` to opt out of any default layout.
* **Placeholder index**: Temporary `app/pages/index.vue` to validate auth flow end-to-end; replaced in Phase 5.
